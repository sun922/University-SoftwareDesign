### **3.1 시나리오** <br>
#### SC-FCM-001 – 학생이 게시글을 작성한다
1. 학생은 로그인 화면(UI-FAU-001)에서 로그인한다.
2. 커뮤니티 게시판 메뉴(UI-FCM-001)로 이동해 게시글 목록을 확인한다.
3. ‘글쓰기’ 버튼을 클릭해 글 작성 화면(UI-FCM-002)으로 이동한다.
4. 제목, 본문을 작성하고 등록 버튼을 누른다.
5. 게시글은 목록 화면으로 리다이렉트되며 상단에 표시된다.

![image](https://github.com/user-attachments/assets/cbdb2766-6986-4cd6-a3e6-5dc71cff976b)



#### SC-FCM-002 – 교수가 게시글을 고정한다
1. 교수는 로그인 후 게시글 목록(UI-FCM-001)으로 이동한다.
2. 자신이 작성한 게시글을 클릭해 상세 화면(UI-FCM-003)으로 들어간다.
3. ‘고정하기’ 버튼을 눌러 글을 상단에 고정한다.

![image](https://github.com/user-attachments/assets/db4e43bf-94a5-4bd0-862d-dfc125b5b679)


#### SC-FNN-001 – 관리자가 공지사항을 작성한다
1. 최고 관리자가 로그인한다.
2. 관리자 메뉴에서 공지사항 등록 화면(UI-FNN-002)으로 이동한다.
3. 제목과 본문을 입력 후 고정 여부를 설정하고 등록한다.
4. 공지사항은 목록(UI-FNN-001)에 최신순으로 노출된다.

![image](https://github.com/user-attachments/assets/78cae201-6bc9-46f2-a25d-62274b5414f8)



#### SC-FPR-001 – 사용자가 교수진 정보를 조회한다
1. 사용자는 ‘교수진 소개’ 메뉴(UI-FPR-001)를 클릭한다.
2. 간략히 정리되어있는 교수 목록에서 상세하게 보고자 하는 교수를 찾는다.
3. 사용자가 특정 교수를 클릭한다.
4. 해당 교수의 세부 정보(전공, 연구실, 이메일 등)를 확인한다.

![image](https://github.com/user-attachments/assets/4f76b9cd-868d-411c-bbee-9d584e8f2acd)



#### SC-FCC-001 – 사용자가 커리큘럼을 조회한다
1. 사용자는 ‘커리큘럼 안내’ 메뉴(UI-FCC-001)를 클릭한다.
2. 학년별 탭/드롭다운에서 원하는 학년을 선택한다.
3. 과목 리스트가 표시되고, 사용자는 과목명을 클릭한다.
4. 연결된 강의계획서 PDF 링크로 이동하거나 다운로드한다.

![image](https://github.com/user-attachments/assets/2814d9b2-6088-415b-8e39-446804b7f8fc)



#### SC-FAM-001 – 최고 관리자가 정적 콘텐츠를 수정한다
1. 최고 관리자가 로그인 후 ‘콘텐츠 관리’ 메뉴(UI-FAM-001)를 클릭한다.
2. 기존 콘텐츠 리스트가 표시되고, 수정할 항목의 ‘편집’ 버튼을 클릭한다.
3. 편집 화면에서 제목, 본문, 이미지 등을 수정한다.
4. ‘저장’ 버튼 클릭 후 콘텐츠가 즉시 반영된다.

![image](https://github.com/user-attachments/assets/bd9e7477-5a6d-4332-a9d9-ff82aa00f24c)


#### SC-FAA-001 – 신규 사용자가 실명 인증 후 회원가입한다
1. 사용자는 로그인 화면에서 ‘회원가입’을 선택해 UI-FAA-001로 이동한다.
2. 이름, 학번, 이메일, 비밀번호를 입력한다.
3. 실명 인증 API를 통해 학교 포털 연동 인증을 시도한다.
4. 인증 성공 시 계정이 생성되며 자동으로 로그인 상태로 전환된다.

![image](https://github.com/user-attachments/assets/aa6fffb0-282c-4e7b-92f6-8a5c237df3b9)
